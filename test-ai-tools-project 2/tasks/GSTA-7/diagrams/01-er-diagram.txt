Entity Relationship Diagram (ASCII)
---------------------------------

Organizations
  id: uuid PK
  name: text
  slug: text (unique)
  plan: text
  quota_limits: jsonb
  metadata: jsonb

Users
  id: uuid PK
  organization_id: uuid FK -> organizations(id)
  email: text
  name: text
  role: text

Projects
  id: uuid PK
  organization_id: uuid FK -> organizations(id)
  name: text
  slug: text

Resources
  id: uuid PK
  organization_id: uuid FK -> organizations(id)
  project_id: uuid FK -> projects(id) (nullable)
  type: text
  name: text
  data: jsonb
  status: text (active|deleted|archived)
  deleted_at: timestamptz (soft-delete)

AuditLogs (append-only)
  id: uuid PK
  organization_id: uuid FK -> organizations(id)
  actor_id: uuid FK -> users(id) (nullable)
  resource_type: text
  resource_id: uuid (nullable)
  action: text
  timestamp: timestamptz
  payload: jsonb (before/after snapshots)

Indexes and constraints (high level)
- users_org_email_idx: UNIQUE(organization_id, lower(email))
- projects_org_slug_idx: UNIQUE(organization_id, slug)
- resources_org_project_created_idx: INDEX(organization_id, project_id, created_at DESC)
- audit_logs_org_res_idx: INDEX(organization_id, resource_type, resource_id, timestamp DESC)

Notes
- Use UUID PKs generated by DB or application (gen_random_uuid()).
- Keep AuditLogs append-only and small payloads when possible (avoid storing large blobs).
