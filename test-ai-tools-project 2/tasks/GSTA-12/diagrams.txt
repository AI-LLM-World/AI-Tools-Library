Diagrams - GSTA-12 Phase 7

Component diagram (ASCII):

  [Browser Admin] --HTTPS--> [Frontend App] --HTTPS--> [Backend API]
                                               |---> [Submissions Store: data/submissions.json]
                                               |---> [Audit Log / Metrics]
                                               |---> [Secrets Manager]

Sequence diagram for programmatic submission:

  Client (script) -> Backend API: POST /api/tools/submit (Authorization: Basic .., X-Idempotency-Key)
  Backend API -> Auth verifier: validate client_id/secret
  alt auth ok
    Backend API -> Validator: validate payload
    alt payload ok
      Backend API -> Storage: atomic append to submissions.json
      Storage --> Backend API: success
      Backend API --> Client: 201 Created {id, status: pending}
    else payload invalid
      Backend API --> Client: 400 Bad Request
    end
  else auth fail
    Backend API --> Client: 401 Unauthorized
  end

Submission state diagram:

  [new] --> (submitted) -> pending
     pending -> approved -> published
     pending -> rejected -> closed
     approved -> published

Data flow notes:
- Submissions land in submissions.json with status pending; approval triggers moving the entry into the published ai_tools.json catalog and updates status.

(End of file)
