PR Checklist for GSTA-40

When opening a PR for the Admin Dashboard User Guide and supporting files, include the following in the PR description:

- Summary of changes (docs + tests + task metadata)
- Related issues (GSTA-40)
- Acceptance criteria checklist from tasks/GSTA-40/issue.md
- Request reviewers: Staff Engineer, Release Engineer, QA Engineer
- Testing steps reviewers should run locally (run integration tests, open docs)

Required before merge:
- Staff Engineer sign-off on endpoint accuracy and screenshots
- QA sign-off on test matrix and successful CI smoke run
- Release Engineer sign-off on deploy checklist
